# 제목(Title)
Agile dead trees 특성 추출 및 아키텍처 결정

# 상태(Status)
수락

# 컨텍스트(Context)
수십 명의 출판사 직원, 수백 명의 저자, ​​수천/수백만 명의 고객: 가용성, 확장성, 성능
저자는 챕터를 출판
검토자는 챕터를 보고, 검토 의견을 작성하고, 검토에 대해 저자에게 알립니다.
저자는 제안된 리뷰 변경 사항을 거부할 수 있습니다.
복사 및 기술 편집을 모두 지원합니다.
고객은 '베타'로 제공되는 도서를 포함하여 온라인으로 도서(전자 양식 또는 종이 양식)를 구매할 수 있습니다.
출판사는 '베타'를 구매한 고객에게 저자의 챕터를 push할 수 있습니다.

#추가 컨텍스트
경쟁업체가 유사한 제품을 제공하고 있기 때문에 비즈니스가 이러한 결정을 내리게 되었습니다. 작가 경쟁이 치열합니다: 가용성, 성능
이는 비즈니스의 출판 측면을 현대화하기 위한 장기 전략의 일부입니다.
책을 출판하는 데 필요한 정보(배포, 로열티, 마케팅)는 이메일로 전송된 Excel 스프레드시트부터 인쇄 시설과의 메인프레임 통합에 이르기까지 여러 서로 다른 시스템에서 나옵니다: 확장성

요구사항에 기술되지 않은 암묵적 특성들
- 보안성
- 신뢰성

# 결정(Decision)
다음 특성들을 주요하게 고려하여 아키텍처를 선택하기로 한다.

- 보안
- 신뢰성
- 가용성
- 성능
- 확장성

이러한 특성을 고려한 결과 마이크로서비스 아키텍처를 선택하기로 결정한다. 솔루션으로는 docker + kubernetes를 사용하고, github workflow로 CI/CD하여 최종적으로 만들어진 docker image를 kubernetes에 배포하는 방식을 취한다. 

# 결과(Consequences)
마이크로서비스 아키텍처를 선택하여 확장성(여러가지 정보를 통합하고 확장), 가용성과 성능면에서 잇점을 얻을 수 있다.
